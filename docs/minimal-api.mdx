## Minimal

### Server API

To use React Server Components in Waku, you need to create a `./src/waku.server.tsx` file and `defineHandlers` to handle server requests.

```tsx
import { unstable_defineHandlers as defineHandlers } from 'waku/minimal/server';
import adapter from 'waku/adapters/default';
import { Slot } from 'waku/minimal/client';
import App from './components/App.js';

const handlers = defineHandlers({
  handleRequest: async (input, { renderRsc, renderHtml }) => {
    if (input.type === 'component') {
      return renderRsc({ App: <App name={input.rscPath || 'Waku'} /> });
    }
    if (input.type === 'custom' && input.pathname === '/') {
      return renderHtml(
        await renderRsc({ App: <App name="Waku" /> }),
        <Slot id="App" />,
        {
          rscPath: '',
        },
      );
    }
  },
  handleBuild: async () => {},
});

export default adapter(handlers);
```

Tips for experimenting:

- Object keys like `App` are called RSC ID.
- `input.type === 'custom'` condition can be omitted if you don't need SSR.

#### `handleBuild`

The `handleBuild` function runs during `waku build`.
Unlike older versions, it does not return build instructions.
Instead, it is an async function that receives build utilities and emits files with `generateFile`.

If you only need dynamic SSR, an empty handler is enough:

```tsx
  handleBuild: async () => {},
```

To prerender the root route in the minimal example:

```tsx
  handleBuild: async ({
    renderRsc,
    renderHtml,
    rscPath2pathname,
    generateFile,
  }) => {
    const rscPath = '';
    const stream = await renderRsc({ App: <App name="Waku" /> });
    const [rscStream, htmlStream] = stream.tee();

    await generateFile(rscPath2pathname(rscPath), rscStream);

    const res = await renderHtml(htmlStream, <Slot id="App" />, {
      rscPath,
    });
    await generateFile('index.html', res.body!);
  },
```

`generateFile` writes files into `dist/public`.
`rscPath2pathname` converts an RSC path such as `''` into the corresponding RSC payload pathname.

### Client API

To render a React Server Component on the client, use the `Root` and `Slot` components from `waku/minimal/client` with the RSC ID to create a wrapper component.

This file should be `./src/waku.client.tsx`.

```tsx
import { StrictMode } from 'react';
import { createRoot } from 'react-dom/client';
import { Root, Slot } from 'waku/minimal/client';

const rootElement = (
  <StrictMode>
    <Root>
      <Slot id="App" />
    </Root>
  </StrictMode>
);

createRoot(document.getElementById('root')!).render(rootElement);
```

The `initialRscPath` prop can be passed to the `Root` Component, overriding the default rscPath which is `""`. You can also re-render a React Server Component with new rscPath.

```tsx
import { useRefetch } from 'waku/minimal/client';

const Component = () => {
  const refetch = useRefetch();
  const handleClick = () => {
    refetch('...');
  };
  // ...
};
```
